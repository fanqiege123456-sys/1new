# GitNetDisk 用户端界面设计文档

## 1. 设计理念
**核心原则**：网盘式操作体验 + GitHub 原生功能融合

- **用户视角**：仓库 = 文件夹，文件操作与本地网盘一致
- **视觉风格**：简洁现代，符合 2026 年设计趋势
- **交互模式**：拖拽为主，点击为辅，减少学习成本
- **响应式**：支持桌面端优先，兼顾平板设备

## 2. 页面结构

### 2.1 整体布局
```
┌─────────────────────────────────────────────────────┐
│ 顶部导航栏（Logo + 用户信息 + 设置）                │
├───────────────┬─────────────────────────────────────┤
│               │                                     │

│（左侧边栏）   │  （主内容区）                       │
```

| 页面名称 | 模块组成 | 核心功能 |
|---------|---------|--------|
| **登录页** | 登录按钮 + 项目介绍 | GitHub OAuth 登录 |

## 4. 交互设计

### 4.1 核心交互流程

**登录流程**：
1. 用户访问应用 → 显示登录页
2. 点击"GitHub 登录" → 跳转 GitHub 授权页
3. 用户授权 → 跳转回调页 → 获取 token
4. 保存 token → 自动跳转主界面

**文件上传流程**：
1. 用户进入目标目录
2. 拖拽文件到浏览器区域
3. 显示上传进度
4. 上传完成 → 自动刷新文件列表
5. 显示成功提示

**仓库创建流程**：
1. 点击"新建仓库"按钮
2. 弹出表单：仓库名称、描述、是否公开
3. 提交 → 显示创建进度
4. 创建完成 → 自动添加到仓库列表

### 4.2 微交互设计
- **拖拽反馈**：文件拖拽到浏览器时，区域高亮显示
- **加载状态**：所有 API 调用显示加载动画
- **操作反馈**：所有操作显示成功/失败提示
- **hover 效果**：按钮和可点击元素有 hover 效果
- **错误处理**：网络错误时显示友好提示，建议检查代理设置

## 5. 视觉设计

### 5.1 色彩方案
- **主色**：黑金组合 - 金色 (#D4AF37) + 深黑 (#0A0A0A)
- **辅助色**：古铜金 (#B8860B)、香槟金 (#F7E98E)、炭灰 (#1A1A1A)
- **功能色**：翡翠绿 (#046307)、琥珀黄 (#FFBF00)、宝石红 (#9B111E)
- **背景色**：纯黑 (#000000)、深灰黑 (#121212)、超浅灰 (#1E1E1E)
- **文本色**：亮金 (#FFD700)、银灰 (#C0C0C0)、浅灰 (#808080)

### 5.2 字体系统
- **主字体**：Playfair Display（标题） + Inter（正文）
- **代码字体**：JetBrains Mono 或 Fira Code
- **字号**：
  - 标题：24px（金色，加粗）
  - 副标题：18px（浅金色）
  - 正文：14px（银灰色）
  - 小字：12px（浅灰色）

### 5.3 组件样式
- **按钮**：圆角 8px，黑金渐变背景，金色边框
- **输入框**：圆角 6px，深色背景 (#1E1E1E)，金色边框
- **卡片**：深色背景 (#121212)，金色边框，微妙发光效果
- **图标**：金色描边，深色背景
- **间距**：遵循 12px 网格系统，更宽敞的布局
- **特效**：微妙的金色光晕、渐变背景、金属质感

### 5.4 高级感细节
- **渐变**：黑到金的微妙渐变背景
- **纹理**：亚光金属纹理叠加
- **光影**：柔和的金色光源效果
- **边框**：1px 金色边框，精致感
- **按钮效果**：hover 时金色光晕增强
- **字体效果**：标题文字轻微阴影，增强立体感

## 6. 响应式设计

### 6.1 断点设置
- **桌面**：≥ 1024px（完整布局）
- **平板**：768px - 1023px（侧边栏可折叠）
- **移动**：< 768px（单栏布局，底部导航）

### 6.2 适配策略
- **桌面端**：完整双栏布局，最佳体验
- **平板端**：侧边栏可收起，通过按钮展开
- **移动端**：单栏布局，仓库列表通过底部导航切换

## 7. 技术实现要点

### 7.1 前端技术
- **框架**：React 18 + TypeScript
- **构建工具**：Vite
- **UI 库**：Ant Design 5 + Tailwind CSS
- **状态管理**：Zustand
- **文件操作**：react-dropzone
- **图标**：@ant-design/icons

### 7.2 性能优化
- **懒加载**：组件和图片懒加载
- **缓存**：文件列表缓存，减少 API 调用
- **防抖**：搜索和输入防抖
- **虚拟滚动**：大文件列表使用虚拟滚动

### 7.3 可访问性
- **键盘导航**：支持 Tab 键导航
- **屏幕阅读器**：ARIA 标签支持
- **高对比度**：支持系统高对比度模式
- **快捷键**：常用操作支持快捷键

## 10. 总结

GitNetDisk 的界面设计遵循以下原则：
- **用户友好**：网盘式操作逻辑，降低学习成本
- **视觉现代**：符合 2026 年设计趋势，简洁美观
- **功能完整**：覆盖 MVP 所有核心功能
- **技术可行**：基于现代前端技术栈，实现难度适中

通过这套设计方案，我们可以打造一个既美观又实用的 GitHub 网盘工具，让用户能够更便捷地使用 GitHub 进行文件存储和管理。
