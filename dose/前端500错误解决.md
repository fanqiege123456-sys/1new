# 前端 500 错误解决方案

## 错误信息
```
GET http://localhost:5173/App.tsx?t=1768977157112 net::ERR_ABORTED 500 (Internal Server Error)
```

## 问题分析
这是 **Vite 开发服务器** 的错误，不是后端 API 的问题。

### 原因
- Vite 的热模块替换（HMR）在尝试重新加载 `App.tsx` 时失败
- 可能是文件被临时锁定、缓存问题，或者 Vite 服务器状态异常

### 不是后端问题
注意 URL 是 `http://localhost:5173/App.tsx`（前端端口），不是 `http://localhost:3000/api/...`（后端 API）

## 解决方案

### 方法 1: 重启前端开发服务器（推荐）
```bash
# 在前端目录
cd git-web

# 停止当前服务器（Ctrl+C）
# 然后重新启动
npm run dev
```

### 方法 2: 清除缓存并重启
```bash
cd git-web

# 删除 node_modules/.vite 缓存
rmdir /s /q node_modules\.vite

# 重新启动
npm run dev
```

### 方法 3: 完全重新安装（如果上述方法无效）
```bash
cd git-web

# 删除 node_modules
rmdir /s /q node_modules

# 重新安装依赖
npm install

# 启动服务器
npm run dev
```

## 验证
重启后，浏览器应该能正常访问 `http://localhost:5173`，不再出现 500 错误。

## 注意事项
- 这个错误与文件上传功能无关
- 后端 API 的修复已经完成，重启后端后文件上传应该能正常工作
- 前端和后端是两个独立的服务，需要分别重启
