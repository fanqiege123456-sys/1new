# GitNetDisk 前端设置指南

## 📦 安装依赖

```bash
cd git-web
npm install
```

## 🚀 启动开发服务器

```bash
npm run dev
```

## 🔧 配置说明

### 1. GitHub Personal Access Token (PAT)

1. 访问 [GitHub Settings > Tokens](https://github.com/settings/tokens)
2. 点击 "Generate new token (classic)"
3. 勾选 `repo` 权限（完整的仓库访问权限）
4. 生成并复制 token
5. 在登录页面输入 token

### 2. 代理配置

#### 方式一：直连模式（默认）
- 不启用代理，直接连接 GitHub API
- 适合网络环境良好的用户

#### 方式二：后端代理模式（推荐国内用户）
1. 在设置页面启用"启用后端代理服务"
2. 配置代理服务器地址，例如：
   - HTTP 代理: `http://127.0.0.1:7890`
   - SOCKS5 代理: `socks5://127.0.0.1:1080`
   - 带认证: `http://user:pass@proxy.example.com:8080`

### 3. 后端 API 配置

前端会通过以下方式调用 GitHub API：

**直连模式**：
```
前端 → https://api.github.com/user/repos
```

**代理模式**：
```
前端 → /api/proxy?target=https://api.github.com/user/repos
      ↓ (请求头: x-proxy-url: http://127.0.0.1:7890)
后端代理服务 → GitHub API
```

## 🎨 功能特性

- ✅ GitHub OAuth 登录（使用 PAT）
- ✅ 仓库列表展示
- ✅ 文件浏览器
- ✅ 拖拽上传文件
- ✅ 创建新仓库
- ✅ 代理配置（支持 HTTP/SOCKS5）
- ✅ 主题切换（浅色/深色）

## 📝 注意事项

1. **Token 安全**：Token 存储在浏览器 localStorage 中，请勿在公共设备上使用
2. **代理配置**：代理地址通过请求头 `x-proxy-url` 传递给后端
3. **文件大小限制**：GitHub API 限制单个文件最大 100MB
4. **API 限流**：GitHub API 有速率限制，建议合理使用
